# 試煉塔改版設計（2026-07-11）

## 一、目標

把單軌爬塔改成 **6 座屬性主題塔、可自由跳關的 1–80 關挑戰**。核心訴求：

1. **自由跳關**：不用逐層慢慢打，任意關直接挑戰。
2. **6 種環境入口**：進去先選一座塔（＝一種天氣或場地），該塔 80 關全程固定那個環境。
3. **等級＝關數**：第 N 關敵人等級就是 N（1–80）。
4. **每 5 倍數關 = 手工強隊、要難打**：16 個 Boss 關（5,10,…,80）用精心搭配的羈絆隊，可重複。其餘關卡隨機生成。
5. **60 關以後的 Boss 隊一定要吃該塔環境**（全隊該塔吃香屬性）；5–55 關能吃最好、不強制。

## 二、玩家體驗與結構

### 6 座主題塔

| 塔 | 環境（全程固定） | 吃香屬性 | 環境效果 |
|---|---|---|---|
| 烈日塔 | 天氣：烈日 | 火 | 火造成傷害 +20%、水 −20% |
| 暴雨塔 | 天氣：暴雨 | 水 | 水造成傷害 +20%、火 −20% |
| 颶風塔 | 天氣：颶風 | 風 | 風造成傷害 +20%、受傷 −10% |
| 湧能塔 | 場地：湧能磁場 | 光 | 光受傷 −15%、集氣 +20% |
| 侵蝕塔 | 場地：侵蝕之地 | 暗 | 暗暴擊 +20%；**非暗每回合流失 10% 最大生命** |
| 沼澤塔 | 場地：迷霧沼澤 | 毒（dot） | 雙方持續傷害 +20% |

> **設計巧思**：60 關後 Boss 全隊走吃香屬性 → 環境同時「加成敵人」又「懲罰玩家的天然剋制」。例如烈日塔火隊吃 +20%，玩家想帶水剋火反被 −20%；侵蝕塔全暗隊讓玩家非暗隊每回合流失血量、被時間追殺。不用灌超大數值就很難。

### 導覽流程

1. 主城 → 試煉塔 → **選塔頁**（6 張塔卡，顯示環境與吃香屬性）。
2. 進塔 → **關卡格 1–80**：可任意點關挑戰（自由跳關）。每格顯示：關數、敵人等級、環境徽章、首通獎、是否已通（✓）、Boss 關標記（★）。
3. 點關 → 預覽敵隊 → 出戰 → 回放 → 勝利首通入帳。

### 獎勵：每關首通獎（每塔獨立）

- 任意關都能打；每關**首次通關**發一次獎，隨關數成長。跳到 80 關打贏只拿 80 關首通獎，不用補 1–79。
- 公式（沿用現制精神）：`gold = 200 + floor×80`、`essence = 10 + floor×4`；Boss 關追加召喚券 `1 + floor(floor/25)`。
- 進度以 **(塔, 關)** 為單位記錄已通，跳關不影響。

## 三、難度模型

### 等級曲線

- `enemyLevel(floor) = floor`（1–80）。
- 玩家滿級 **Lv60** → **61–80 關敵人超過玩家等級上限**，是刻意的終局牆：只能靠 5★、羈絆、環境吃香屬性硬吃。**（已與使用者確認採此方向）**

### 星級曲線

- `enemyStars(floor) = clamp(floor(floor / 12), 0, 5)`：12 關 1★ … 60 關 5★。60 關後全 5★。
- Boss 關 **額外 +1★**（上限 5★），比同段路關再硬一階。

### Boss 溢價（每 5 倍數關）

「不好打」＝三層疊加：

1. **羈絆隊**：用卡池真實的隊伍技／光環／觸發（隨機隊沒有）。
2. **數值溢價**（三圍乘算）：
   - 5–25 關 ×1.15
   - 30–55 關 ×1.25
   - 60–80 關 ×1.35
3. **環境吃香**：5–55 關盡量同屬性；**60–80 關全隊清一色吃香屬性**（沼澤塔＝全隊 dot 載體）。

### 非 5 倍數關（路關）

- 隨機生成，**偏向該塔吃香屬性**（≥半數同屬）、至少 1 坦。
- 等級＝關數、星級照曲線、**無溢價**。定位是快速過路。

## 四、精心隊資料表

站位規則：`melee → 前排(1–3)`、`ranged / support → 後排(4–6)`；前排滿了溢位到後排。以下每隊 6 名，格式 `前排 ｜ 後排`。卡片用中文名（附 id）。

低段（5,10,15,20,25）：A/B 兩隊輪替。中段（30,35,40,45,50,55）：A/B/C 三隊輪替。高段（60,65,70,75,80）：五隊各一，**全吃香屬性**，80 關為壓軸展示隊。

## 六、測試計畫

1. **等級/星級/溢價**：`enemyLevel(80)===80`、`enemyStars(60)===5`、Boss 三圍＝基礎×溢價（分段驗）。
2. **精心隊合法性**：每塔每 Boss 關建出 6 名有效 Unit、站位不衝突。
3. **60–80 屬性約束**：高段隊全員＝themeElement（沼澤塔除外，改驗全員技能帶 dot）。以測試掃 `towerTeams.js` 靜態驗證，避免手滑放錯屬性。
4. **首通獎**：同 (塔,關) 只入帳一次；跳關到高關直接拿該關獎。
5. **確定性**：同 (塔,關) 路關永遠同隊。
6. **存檔遷移**：舊 `tower.floor` → 新結構不遺失既有進度。
