# UI 視覺改版規格書（給前端工程師）

> 版本：2026-07-05／發包範圍：**非戰鬥 DOM 介面全部頁面的視覺層**
> 戰鬥畫面（PixiJS canvas）不在此次範圍，但戰鬥資訊 HUD（DOM overlay）包含在內。

---

## 1. 我想要的感覺（Design Direction）

**一句話**：《劍與遠征》(AFK Arena) 那種**溫暖、精緻、有手工質感的中古奇幻**——羊皮紙告示、木刻框、黃銅鉚釘、金綢帶，但畫面乾淨明快、不髒不灰。

目前版本的問題（要修正的方向）：

1. **顏色層次太單一（整體土黃）**——現在整個畫面從底到面板到按鈕都是相近的黃棕色，糊成一片。正確做法是**拉開三個層次的對比**：
   - **底層（最暗）**：深胡桃木／深皮革，接近黑的暖棕（`#2a1c10` 附近），可以帶木紋紋理
   - **內容層（最亮）**：象牙色羊皮紙（偏白的 `#f2e4c2` 而非土黃 `#d6ba8a`），深褐墨字，內容列與底層之間要有明顯明度差
   - **點綴層（彩度）**：金色只給主行動與獎勵；另外引入**寶石色點綴**（藍寶石 CTA、翡翠綠成功、緋紅警示）避免滿版暖色的單調
2. **質感不足**——現在是純 CSS 漸層，希望升級成**貼圖質感**：木紋、紙纖維、金屬浮雕邊框（見 §6 素材清單，可用 9-slice）
3. **裝飾語彙**——AFK 風格的特徵元素：撕邊羊皮紙、木刻橫幅（標題）、六角切角綢帶鈕、黃銅角釘、盾形紋章框。用在正確的位置，不要每個元素都堆滿。

參考基準：AFK Arena 的公會頁／推薦公會頁（用戶提供過截圖，向 PM 索取）。

## 2. 技術架構與合約（你能動什麼、不能動什麼）

- 前端是 **Vite + 原生 JS + 純 CSS**，無框架。UI 由 JS 以固定 class 名產生 DOM。
- **合約：JS 不動、class 名不動**——你只改三個 CSS 檔（`src/style.css`、`src/redesign.css`、`src/social.css`）與新增素材檔（`public/assets/ui/`）。若必須加 wrapper 或改 DOM 結構，開需求單回來討論。
- 版面配置（layout）**維持現狀**：各頁的欄位位置、格線、資訊架構都已定案，這次只做視覺層（配色、質感、邊框、字體層級、細節動效）。
- 螢幕：**橫向**手遊比例（16:9 為主），`html { font-size: clamp(9px, 2.05vmin, 16px) }` 全域縮放，尺寸用 `rem`。
- 一切互動元素已有 `.pressable`（按壓縮放回饋），保留。

## 3. Design Tokens API（CSS 變數＝合約）

全部定義在 `src/style.css` 的 `:root`。**你可以改變數的值，不要改變數名**（JS 與三個 CSS 檔大量引用）。以下是變數清單與我期望的方向：

```css
:root {
  /* ── chrome：底與面板（要暗、要有質感，拉開與紙的對比） ── */
  --bg:          /* 主底色：深胡桃木，建議 #332413~#3a2a16 */
  --bg-deep:     /* 更深的底（頁面漸層終點/body）：#241708 附近 */
  --panel:       /* 皮革面板：#4a3520 附近 */
  --panel-2:     /* 面板亮部：#5a4228 附近 */
  --border:      /* 黃銅邊框：#8a6a3f，重要框可亮到 #b08a4f */
  --border-soft: /* 次要邊框 */
  --text:        /* chrome 上的字：象牙白 #f4e8cd */
  --dim:         /* chrome 上的次要字 */

  /* ── 羊皮紙（內容列/卡片：淺底深字） ── */
  --paper:       /* 紙面亮部：建議提亮到 #f2e4c2（現在太土黃） */
  --paper-2:     /* 紙面中間調 */
  --paper-deep:  /* 紙面暗部/邊緣 */
  --ink:         /* 紙上的墨字：#4a3018 */
  --ink-dim:     /* 紙上次要字 */
  --paper-line:  /* 紙的描邊 */

  /* ── 強調色（稀缺使用） ── */
  --gold:        /* 主行動金：#f2c95c */
  --gold-deep:
  --blue:        /* 寶石藍 CTA（前往/創建類）：#7fc4e8 */
  --hp:          /* 成功/生命綠 */
  --danger:      /* 警示紅 */

  /* ── 元素色（卡面/戰鬥共用，勿改值——pixi 端寫死同色） ── */
  --fire --wind --water --light --dark
}
```

**新增權**：你可以新增變數（例如 `--wood-texture`、陰影 token），命名用 `--ui-*` 前綴避免衝突。

## 4. 元件庫 API（依重要度排序）

每個元件列出：class、DOM 結構、狀態、期望視覺。

### 4.1 羊皮紙卡（`.paper`／`.gd-paper`）
- **用在**：所有內容列（公會列、成員列、商店卡、留言板…），是整個風格的主角。
- **DOM**：單一 `div`，內容自由。
- **期望**：象牙紙面（提亮！）＋上下撕邊或毛邊質感（可用 border-image 9-slice 或 mask）；頂部 1px 高光、底部壓深；hover 微浮起。目前純漸層版太平。
- 內文字色一律 `--ink`／`--ink-dim`（JS 已在元件內指定 class，紙上不會出現淺色字）。

### 4.2 六角綢帶鈕（`.gd-hexbtn`，修飾詞 `.stone` `.blue` `.mini` `.wide` `.done`）
- **用在**：公會頁所有動作。**希望推廣成全遊戲的標準按鈕**（可加新 class 別名如 `.ui-ribbon`，但舊 `.btn`/`.btn-gold` 也要一起換皮，兩者並存）。
- **期望**：金屬綢帶質感（上亮下暗、邊緣高光、按下內凹）；`clip-path` 六角切角維持；`.stone`＝石灰銀（次要）、`.blue`＝寶石藍（前往/創建）、`.done`＝去飽和壓灰。

### 4.3 木刻橫幅標題（`.gd-banner`、`.page-title`）
- **期望**：`.gd-banner` 做成雕花木牌（可素材化）：深木底、描金字、兩端裝飾切角。`.page-title`（每頁左上標題字）維持金字＋底部金飾條，但字體可加襯線感（見 §7 字體）。

### 4.4 盾徽框（`.gd-shield`，`.big` 變體）
- **用在**：公會徽章。**期望素材化**：金屬包邊的盾形底座，中央放紋章圖（目前是 emoji 佔位，素材清單含 6~8 個紋章 SVG）。

### 4.5 通用按鈕（`button`、`.btn`、`.btn-gold`、`.btn-danger`、`.icon-btn`）
- 全遊戲既有按鈕。換皮方向同 4.2（黃銅邊、皮革底／金綢帶）。

### 4.6 藥丸與徽章（`.pill`、`.dot`、`.hs-tag` 系列）
- `.pill`：頂欄貨幣（金幣/券/鑽）。期望：黃銅包邊小膠囊、圖示凹槽感。
- `.hs-tag`（角色詳情的職業/屬性/種族/系列標籤）：`.cls`（金框）`.el`（元素色框）`.race`（綠框）`.series`（金字）——保持語義色，質感升級。
- `.dot`：紅點通知，保留。

### 4.7 面板／彈窗（`.hs-panel`、`.ov-*` 系列、`.hx-rail`）
- `.hs-panel`：角色詳情右欄。期望：深皮革底＋黃銅描邊＋四角鉚釘。
- 彈窗（`.ov-playercard`、`.ov-arena-board` 等，共用 `.ov-title`、`.ov-close`）：同語言；`.ov-close` 做成銅釦叉。
- `.hx-rail`：英雄頁左側篩選欄——深色半透明改成皮革條。

### 4.8 卡面（`.cardframe` 系列——謹慎處理）
- `.cardframe-inner`（內金線）、`.cardframe-elbadge`（左上屬性寶石）、`.cardframe-clsbadge`（右下職業章）、`.cardframe-lvl`、`.cardframe-name`、`.cardframe-stars`。
- 立繪是程式生成 SVG（會換成 AI 圖），**卡框可素材化**：木質雕花外框＋金屬內線（9-slice）。徽章底座可做成寶石鑲嵌。

### 4.9 首頁大廳（`.hub-*` 系列）
- `.hub-tl`（玩家名牌）、`.hub-cur`（貨幣列）、`.hub-feat/.fsq`（右上功能格）、`.hub-adv`（右下出征大鈕）、`.hub-dock/.hub-dk/.dksq/.dkl`（底部功能碼頭）。
- 期望：功能格與碼頭做成**木質吊牌／銅飾圖示格**；出征鈕是全頁最強視覺（金綢帶大鈕＋微呼吸光）。
- 背景（`#screen-home` 黃昏山谷漸層＋`.hub-hills` 山影）可整體重繪成插畫背景圖（素材）。

### 4.10 戰鬥 HUD（DOM overlay：`.bo-*` 系列）
- `.bo-cb`（倍速/快轉圓鈕）、`.bo-env`（環境徽章）、頂部隊伍血量匯總、`.hub-*` 不含。canvas 內部不動。
- 期望：深色半透明底＋黃銅邊，維持高對比可讀性（戰鬥上方不能搶戲）。

## 5. 頁面優先序與驗收畫面

改版順序建議（每頁完成後截 1280×720 對比圖驗收）：

1. **首頁大廳**（第一印象）
2. **公會頁**（已有一版新風格，補質感到位＝全遊戲樣板）
3. **英雄頁＋角色詳情**（使用時間最長）
4. **隊伍頁**
5. 召喚、商店、競技場、好友、試煉塔
6. 彈窗全家（任務/簽到/掛機/名片/確認 sheet/toast）

**驗收標準**：
- 三層對比成立：暗木底 ↔ 亮紙內容 ↔ 金/寶石點綴（截圖轉灰階仍能分清層次）
- 紙上深字對比 ≥ WCAG AA（4.5:1）；chrome 上象牙字同標準
- 無版面位移（layout 不變，只換皮）
- 60fps：hover/按壓動效只用 transform/opacity/filter
- `deck-item` 的 `content-visibility: auto` 與各捲動容器行為保留

## 7. 字體與動效

- **字體**：中文維持系統字（PingFang TC/微軟正黑），但**標題層**（`.page-title`、`.gd-banner`、`.ov-title`、卡名）加 `letter-spacing` 與更重的字重對比；數字一律 `font-variant-numeric: tabular-nums`（多處已設，保持）。若引入自訂標題字型（如思源宋體 subset），需打包進 repo，不可外連 CDN。
- **動效節奏**：進場 stagger 已由 JS 控制（`staggerIn`），CSS 只補 hover（浮起 2~4px＋陰影）與按壓（已有 `.pressable`）。金色主鈕可加 3~4s 週期的微光掃過（`@keyframes` + mask，低調）。
- **禁止**：全頁面級的持續動畫（耗電）、layout 抖動、`transition: all`。

## 8. 已知地雷（改壞會炸的東西）

1. `.deck-item` 的 `content-visibility: auto` + `contain-intrinsic-size`——英雄頁效能命脈，只能改視覺屬性。
2. `.screen`/`.screen.active` 的 display 切換、`#overlay-root` 層——router 依賴。
3. `.pressing::after` 按壓高亮由手勢系統動態掛，保留 selector。
4. 元素色變數（`--fire` 等）與 PixiJS 端寫死同色值，改了會跟戰鬥棋子對不上。
5. `.hs-stats .v small.up`（升級增量藍字）、`.st-mile.on`（星級里程碑點亮）這類「狀態語義色」要保留可辨識性。
6. 三個 CSS 檔有少量 `!important`（`.paper-input`），動之前先全域搜尋。
